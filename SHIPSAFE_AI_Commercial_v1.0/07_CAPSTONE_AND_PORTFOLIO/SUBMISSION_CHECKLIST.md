# Capstone Submission Checklist

- [ ] One narrow problem and user are clearly stated.
- [ ] The demo proves a real outcome, not only a UI.
- [ ] Mock/placeholder behavior is disclosed.
- [ ] Setup instructions are reproducible.
- [ ] Secret values and private data are absent.
- [ ] Permissions and safety boundaries are explained.
- [ ] Evaluation scenarios include at least one failure case.
- [ ] Known limitations are explicit.
- [ ] Screenshots/video contain no sensitive information.
- [ ] Repository and public links were reviewed as a stranger.
