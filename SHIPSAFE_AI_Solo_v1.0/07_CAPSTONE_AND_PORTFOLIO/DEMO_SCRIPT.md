# Three-Minute Demo Script

## 0:00-0:20 - Problem
Who experiences the problem and why it matters.

## 0:20-0:40 - Verified promise
The exact outcome the system produces.

## 0:40-1:40 - Live primary journey
Use a prepared realistic input. Explain only the decision points that matter.

## 1:40-2:10 - Evidence
Show durable state, output, test result, or audit record proving the action was real.

## 2:10-2:35 - Safety/failure behavior
Demonstrate one permission boundary or safe failure.

## 2:35-3:00 - Architecture and next step
Show the core components, current limitation, and credible next improvement.
